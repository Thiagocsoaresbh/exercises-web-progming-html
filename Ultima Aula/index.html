Fundamentos de Web

Nesta seção explicamos história, modelo cliente/servidor, navegadores e protocolos, e o papel de front-end e back-end em uma aplicação web.

História resumida da Web

A Web surgiu no início dos anos 1990 como um sistema de documentos interligados por hiperlinks. Navegadores solicitam documentos a servidores usando protocolos como HTTP, e recebem respostas (HTML, CSS, JS, imagens). Com o tempo, navegadores e padrões evoluíram (HTML4, XHTML, HTML5), habilitando experiências ricas e aplicações completas no navegador.

Cliente/Servidor

O navegador é o cliente que envia uma requisição HTTP a um servidor. O servidor processa (dinamicamente ou não) e responde com um status e um corpo (HTML, JSON, etc.). O cliente então renderiza ou processa a resposta. Esse ciclo se repete a cada navegação ou chamada a uma API.

Navegadores e Protocolos (HTTP/HTTPS)

  • Navegadores: Chrome, Firefox, Safari, Edge. Renderizam HTML/CSS, executam JS.
  • HTTP: protocolo de aplicação sem estado. Métodos típicos: GET, POST, PUT, DELETE.
  • HTTPS: HTTP sobre TLS. Criptografa o tráfego e garante integridade e autenticidade.
  • URLs: identificam recursos (esquema, host, caminho, query).

Front-end e Back-end

Front-end é o que roda no navegador (HTML, CSS, JavaScript) e lida com UI/UX. Back-end roda no servidor (ex.: Node.js, Java, .NET, PHP, Python), cuida de regras de negócio, persistência e APIs. As camadas se comunicam via HTTP/HTTPS trocando JSON/HTML.

Estrutura básica de um site

Um site costuma ter páginas organizadas com cabeçalho, navegação, conteúdo principal, barras laterais e rodapé. Em HTML5 representamos isso com tags semânticas específicas (abaixo).

HTML: Estrutura do Documento

A estrutura mínima é composta de <!DOCTYPE>, <html>, <head> e <body>. Abaixo exibimos um exemplo didático: 

<!DOCTYPE html>
<html lang="pt-BR">
  <head>
    <meta charset="UTF-8">
    <title>Título da Página</title>
  </head>
  <body>
    <h1>Olá, mundo!</h1>
  </body>
</html>

O navegador cria o DOM conforme encontra essas tags, montando a árvore de nós em memória.

Evolução até o HTML5 e Novidades

Comparado ao HTML4/XHTML, o HTML5 trouxe:

  • Semântica: <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>.
  • Multimídia nativa: <audio> e <video> sem plugins externos.
  • Formulários melhores: novos types e atributos (email, url, date, required, pattern...).
  • API de Canvas/SVG e melhor integração para gráficos e vetores.
  • DOCTYPE simplificado e <meta charset> direto.

Observação: APIs como localStorage, history, geolocation fazem parte do ecossistema Web Platform (JavaScript), não do HTML em si, mas nasceram na mesma geração de padrões.

Notas

  • Elementos HTML são declarativos; JavaScript é imperativo e manipula a árvore do DOM.
  • Scripts síncronos sem defer/async bloqueiam o parser até executarem.
  • CSS externo pode atrasar a primeira pintura; crie CSS crítico inline quando necessário.
